# ADR-002: Supabase

**Status:** Accepted  
**Date:** 2026-06-28  
**Last Updated:** 2026-06-28

## Context

Need auth, database, and storage without building a custom backend

## Decision

Use Supabase as the backend-as-a-service platform

## Consequences

Faster MVP than building custom backend; built-in Auth + PostgreSQL + RLS + Storage
