# ADR-010: UUID Primary Keys

**Status:** Accepted  
**Date:** 2026-06-28  
**Last Updated:** 2026-06-28

## Context

Need non-sequential identifiers for security and Supabase compatibility

## Decision

Use UUID v4 as primary keys for all database tables

## Consequences

Non-sequential prevents ID enumeration; Supabase-native; RLS-safe
